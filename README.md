# MAC-IP 自动配置工具

根据网卡MAC地址自动匹配并设置IP地址、子网掩码、网关、DNS和计算机名称。

## 环境要求

- Windows 10/11 或 Windows Server 2016+
- PowerShell 5.1+
- **管理员权限**（设置IP和计算机名需要）

## 文件结构

```
mac-ip-config/
├── Set-NetworkConfig.ps1   # 主程序（PowerShell脚本）
├── Set-NetworkConfig.bat    # 启动器（双击以管理员运行）
└── config.csv               # 配置文件
```

## 配置说明

### config.csv 格式

```csv
Name,IP,Sub,Gateway,DNS1,DNS2,Mac
"STU1104-11","192.168.1.11","255.255.255.0","192.168.1.1","8.8.8.8","8.8.4.4","a0ad9f9886e6"
"STU1104-12","192.168.1.12","255.255.255.0","192.168.1.1","8.8.8.8","8.8.4.4","b0ad9f9886e7"
```

| 字段 | 说明 | 示例 |
|------|------|------|
| Name | 计算机名称 | STU1104-11 |
| IP | IP地址 | 192.168.1.11 |
| Sub | 子网掩码 | 255.255.255.0 |
| Gateway | 默认网关 | 192.168.1.1 |
| DNS1 |首选DNS | 8.8.8.8 |
| DNS2 | 备用DNS | 8.8.4.4 |
| Mac | 网卡MAC地址（无冒号） | a0ad9f9886e6 |

> **注意**：MAC地址不能有冒号或横线，如 `a0:ad:9f:98:86:e6` 应写成 `a0ad9f9886e6`

## 使用方法

1. 编辑 `config.csv`，添加需要匹配的MAC地址和对应配置
2. 双击 `Set-NetworkConfig.bat`
3. 弹出UAC管理员权限确认，点击"是"
4. 程序自动检测MAC并配置

## 运行流程

```
[1] 读取配置文件
    ↓
[2] 获取本机网卡MAC地址
    ↓
[3] 匹配CSV中的MAC地址
    ↓
[4] 设置网络配置（IP、子网、网关、DNS）
    ↓
[5] 设置计算机名称（如需要）
```

## 注意事项

- 必须以**管理员身份**运行
- 设置计算机名称后需要**重启生效**
- 程序会自动排除虚拟网卡（VMware、VirtualBox、Hyper-V等）
- DNS字段可留空
- 首次运行建议先备份当前网络配置

## 故障排除

| 问题 | 解决方法 |
|------|----------|
| 找不到配置文件 | 确保 `config.csv` 与脚本在同一目录 |
| 未找到匹配的MAC | 检查CSV中MAC地址是否正确（无分隔符） |
| IP设置失败 | 确认该网卡未连接其他网络 |
| 权限不足 | 右键"以管理员身份运行" |
